Heuristics for Interface Analysis

Visibility of System Status

Elite displays most of the vital system information on the main front window, such as ship shield status, power distribution, ship health and comms etc.. However there is still quite a lot of vital information that is rather hard to access quickly.

User Control and Freedom

Elites UI does not have much control and freedom, for example if the user presses the "eject all cargo" key, there is no confirmation.

Recognition Rather than Recall

This is where Elites UI could have a lot of improvements to it. For example:

This image shows the shield status as a set of rings around the ship, this is not clear as to what it actually does.

Flexibility and Efficiency of Use

The navigation of the UI is fairly simple, There are a set of tabs at the top of the UI and each tab contains a set of information about a specific part of the game. All the UI navigation can be done via the keyboard, as there is no mouse in the game. This can be confusing for new players as navigating the UI can be challenging without a mouse. Furthermore some of the key functionality of the game can be quite hard to find and activate, for example the Landing Gear and External Lights on the far tab, as the image below shows.

Aesthetic and Minimalist Design

Most of Elites UI is kept fairly clean, allowing most of the players vision to be focused on the game. Some of the tabs could be combined because I find that I spend a lot of time cycling through the tabs to try and find the right one.

Consistency and Standards
The UI has consistant design elements,
Help and Documentation
There is little in the way of help within the actual game, and there is a steep learning curve for a lot of the elements
